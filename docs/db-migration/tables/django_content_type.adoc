Strong evidence supports the following interpretation:

* `django_content_type` was **not migrated as business data**

== Conclusion
This table is a **framework metadata registry, not a domain-data migration table**:

* schema is unchanged
* uniqueness/integrity rules are preserved
* numeric IDs are not semantically stable across environments
* old/new row-set differences reflect different installed app/model registries
* all content types are valid and used in permissions on each side
* therefore this table should be documented as **environment-regenerated Django metadata**, not as a 1:1 migrated table
