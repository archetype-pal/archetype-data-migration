== Migration mapping logic

=== Position vocabulary
* `symbols_structure_position` is populated from `digipal_aspect` (IDs preserved, 17 rows).
* Therefore `position_id` in `annotations_graph_positions` corresponds directly to `old.aspect_id`.

=== Graph ID mapping (critical difference)
The new model links positions to **annotation graphs** (`annotations_graph`), not to `digipal_graph`.
The effective mapping used is:

* `old_graph_id = digipal_graph_aspects.graph_id`
* Find `digipal_annotation` row where `digipal_annotation.graph_id = old_graph_id`
* `new_graph_id = digipal_annotation.id`
* Insert `(new_graph_id, position_id = aspect_id)`

This implies: old graph-aspect links are only migratable for graphs that appear in `digipal_annotation.graph_id`.
