= public.digipal_hand (old) → public.scribes_hand (new)

== Summary
* Rows: 696 → 696 (complete transfer; IDs preserved)
* Major schema remodel (only 4 shared columns: `id`, `item_part_id`, `scribe_id`, `script_id`)
* New required semantic fields introduced: `name`, `place`, `description`
* New normalized date FK introduced: `date_id` → `common_date`
* Critical transformation fully verified: old NULL `scribe_id` values were migrated to sentinel `-1` (valid FK in `scribes_scribe`)
* `item_part_id`, `script_id`, and `date_id` (vs old `assigned_date_id`) are preserved exactly by row ID
* In this dataset, `place` is a whitespace placeholder for all rows, and `description` is blank for most rows (691/696)

=== `description` derivation status
Tested direct mappings from old hand text fields (`label`, `display_note`, `internal_note`, `comments`) do not explain the new `description` values.
Observed outcome:

* 691 rows have blank/placeholder `description`
* 5 rows contain non-blank HTML-rich content
* Non-blank values are not explained by simple direct equality/fallback rules against tested old `digipal_hand` fields

Conclusion:

* `description` appears to be mostly placeholder/default in this dataset, with a small number of curated or separately transformed values.
* The source of those 5 non-blank descriptions is not evidenced by same-row `digipal_hand` text columns alone.

=== `place` field behavior
Observed outcome:

* `place` is NOT NULL in the new schema
* all 696 rows contain blank/whitespace values

Conclusion:

* In this dataset, `place` is populated with a placeholder blank/whitespace string rather than migrated place names.

== Conclusion
This table migrated completely at the entity level (all IDs preserved) but underwent substantial schema redesign and controlled transformation.
The key documented migration behaviors are:

* exact carry-over of `item_part_id`, `script_id`, and `assigned_date_id` → `date_id`
* strict NOT NULL enforcement for `scribe_id` via sentinel mapping (`NULL` → `-1`)
* introduction of new required text fields with placeholder/default population (`place`, mostly `description`)
* exact derivation of \name` from old `label` (696/696); old `display_label` unused for this mapping`
