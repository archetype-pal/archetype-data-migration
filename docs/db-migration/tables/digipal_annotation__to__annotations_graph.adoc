= public.digipal_annotation (old) → public.annotations_graph (new)

== Summary
* Rows: 24,584 → 24,584 (complete transfer; IDs preserved)
* 1:1 migration by `id` (no missing old IDs; no extra new IDs)
* Shared column set is minimal (`id` only); migration is a major schema redesign rather than a simple column carry-over
* New required FK fields are fully populated and valid:
** `item_image_id` is an exact carry-over of legacy `image_id` for all 24,584 rows
** `allograph_id` is exactly derived from legacy graph linkage on graph-backed rows (`digipal_annotation.graph_id -> digipal_graph.idiograph_id -> digipal_idiograph.allograph_id`)
** `hand_id` is exactly derived from legacy `digipal_graph.hand_id` on graph-backed rows
* `annotation_type` is a semantic normalization of legacy `type`:
** legacy `NULL` → new `'image'`
** legacy `'text'` → new `'text'`
** legacy `'editorial'` → new `'editorial'`
* `annotation` (JSONB) is a new normalized annotation payload:
** valid JSONB in all 24,584 rows
** matches parseable legacy `geo_json` exactly where legacy `geo_json` is valid JSON (4,279/4,279)
** strongly linked to legacy vector data (`annotation` text contains old `vector_id` in 24,571 rows)
* Legacy audit/editorial/geometry helper columns and several old FKs were removed or folded into the new model

== Constraints and indexes

=== Constraints
Old:

* Primary key on `id`
* Unique constraint on `graph_id`
* FKs to:
** `digipal_image(id)` via `image_id`
** `digipal_status(id)` via `status_id`
** `digipal_allograph(id)` via `before_id`
** `digipal_graph(id)` via `graph_id`
** `digipal_allograph(id)` via `after_id`
** `auth_user(id)` via `author_id`

New:

* Primary key on `id`
* FKs to:
** `symbols_structure_allograph(id)` via `allograph_id`
** `scribes_hand(id)` via `hand_id`
** `manuscripts_itemimage(id)` via `item_image_id`

Observation:

* The new schema removes legacy direct links to status/author/before/after/graph and replaces them with a normalized annotation payload plus required domain FKs (`allograph_id`, `hand_id`, `item_image_id`).
* Legacy `UNIQUE(graph_id)` disappears because `graph_id` is no longer stored in the new table.

=== Indexes
Old:

* PK index
* Unique index on `graph_id`
* indexes on `image_id`, `status_id`, `before_id`, `after_id`, `author_id`
* indexes on `clientid` / `type` (+ pattern indexes)

New:

* PK index
* indexes on `allograph_id`, `hand_id`, `item_image_id`
* index + pattern index on `annotation_type`

Observation:

* Indexing shifts from legacy annotation workflow metadata (`graph_id`, `status_id`, `clientid`, etc.) to the new normalized access paths (`item_image_id`, `hand_id`, `allograph_id`, `annotation_type`).
