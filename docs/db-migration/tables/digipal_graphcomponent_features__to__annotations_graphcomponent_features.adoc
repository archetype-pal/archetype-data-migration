* Pair values are exact on all retained rows (13)
* Comparable graphcomponent usage counts are exact (14c)

Therefore:

* The bridge-table delta is explained by omission/filtering of a subset of graphcomponents (and their feature links), not corruption of retained bridge relationships.

== Conclusion
`annotations_graphcomponent_features` is a structurally equivalent bridge table that preserves the old bridge semantics for the rows it includes, but it is **not a complete migration** of `digipal_graphcomponent_features`.

What is preserved exactly:

* schema shape (same logical columns)
* pair uniqueness semantics
* retained row identities by `id`
* retained `(graphcomponent_id, feature_id)` values
* FK validity in the new schema

What changed:

* the new table contains a strict subset of the old bridge rows (2694 of 3367; 673 missing)
* the missing rows are consistent with a reduced migrated graphcomponent population, which in turn lowers feature usage counts for many features
