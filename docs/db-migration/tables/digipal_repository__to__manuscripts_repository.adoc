== Schema redesign observations

The migration is not a simple column-for-column copy. It preserves identity (`id`) and core label (`name`) while simplifying the repository model:

* normalized foreign-key relations removed from the repository row
* textual descriptive fields added (`label`, `place`, `type`)
* legacy flags and audit metadata dropped
* self-referential hierarchy (`part_of_id`) not represented in the new table schema (at least not directly in this table)

This indicates a deliberate denormalization / presentation-oriented redesign in the new manuscripts model.

== Conclusion
This is a complete `id`-preserving migration with exact `name` preservation and preserved downstream usage via current-item references.

What is proven from the provided checks:

* row counts match (9 → 9)
* no missing/extra IDs
* `name` is exact by `id` (9/9)
* shared column length/nullability constraints introduce no blockers
* repository usage counts and FK integrity are preserved in current-item tables

What changed structurally:
* the old repository table’s FK-rich normalized model was replaced by a smaller flattened schema (`label`, `place`, `type`) with only a primary key index/constraint retained.
