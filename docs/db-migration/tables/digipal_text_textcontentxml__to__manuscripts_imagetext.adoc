= public.digipal_text_textcontentxml (old) → public.manuscripts_imagetext (new)

== Summary
* Rows: 972 → 899 (**73-row reduction**)
* Migration is **not ID-preserving** (rekeyed): 175 old IDs missing in new; 102 new IDs not in old
* The row-count reduction is fully explained by filtering: **all 73 excluded old rows have `content IS NULL`**
* Strong semantic validation: the multiset of **(content, status, type)** matches **exactly** between:
  - old rows where `content IS NOT NULL` (899 rows) and

* `status` is migrated as a label from lookup `digipal_text_textcontentxmlstatus.name`
* `type` is migrated as a label from `digipal_text_textcontent.type_id → digipal_text_textcontenttype.name`
* Old `last_image_id` is **NULL for all rows** (not used); new `item_image_id` is fully populated and referentially valid
* `language` is blank for all new rows and old shows **no language links** for the migrated subset
* Item-part coverage matches exactly on both sides: **486 distinct item_parts** represented
