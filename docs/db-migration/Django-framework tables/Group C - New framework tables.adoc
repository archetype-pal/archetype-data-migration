This report documents framework/runtime tables that exist in the new database but do not have an old-table migration counterpart in `old`.

For this dataset, Group C contains:

* `public.authtoken_token`

== Interpretation

`authtoken_token` should be treated as a **new framework/runtime table**, not as migrated project data.

Key reasons:

* it has no legacy counterpart in the old schema set under review
* it belongs to Django REST Framework token authentication infrastructure
* its row count is very small (`1` row), which is consistent with environment/bootstrap usage rather than legacy-data migration
* its only dependency is `auth_user`, meaning its contents are operational credentials tied to users in the new environment
